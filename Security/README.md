Problem 1: Phishing email received

1. Do not click links — Prevents accidental compromise from malicious URLs.
2. Check sender address — Fake domains often mimic legitimate ones.
3. Hover over links — Reveals the true destination before clicking.
4. Look for urgency or threats — Phishing often pressures users to act quickly.
5. Check for spelling errors — Many phishing emails contain poor grammar.
6. Report email to IT/security — Helps organization track threats.
7. Delete the email — Removes risk from inbox.
8. Block sender — Prevents future phishing attempts.
9. Inform the user — Educates them to avoid similar attacks.
10. Escalate if widespread — Multiple users receiving it may indicate a campaign.

Problem 2: Antivirus detects a threat

1. Do not ignore alert — Immediate action prevents further infection.
2. Run full system scan — Identifies all potential threats.
3. Quarantine the file — Isolates the threat safely.
4. Remove the threat — Deletes malicious files.
5. Restart computer — Ensures removal is complete.
6. Update antivirus definitions — Ensures detection of latest threats.
7. Check recent downloads — Identifies infection source.
8. Inform user — Prevents repeat incidents.
9. Document incident — Maintains audit trail.
10. Escalate if persistent — Indicates deeper infection.

Problem 3: Suspicious account activity

1. Reset password immediately — Prevents unauthorized access.
2. Log out all sessions — Removes active attackers.
3. Enable MFA — Adds extra security layer.
4. Review login history — Identifies unusual access.
5. Check IP addresses — Detects suspicious locations.
6. Scan device for malware — Infection may have caused breach.
7. Inform user — Makes them aware of compromise.
8. Report to security team — Required for investigation.
9. Document incident — Maintains records.
10. Monitor account — Ensures no further suspicious activity.

Problem 4: Firewall blocking application

1. Verify application — Ensure it is safe and legitimate.
2. Check firewall settings — Identify blocked ports or rules.
3. Temporarily disable firewall (test only) — Confirms if firewall is cause.
4. Allow application through firewall — Creates exception rule.
5. Restart application — Applies new rules.
6. Check network profile — Public networks may block more traffic.
7. Update application — Fixes compatibility issues.
8. Check antivirus — May also block traffic.
9. Document change — Important for auditing.
10. Escalate if needed — Required for sensitive applications.

Problem 5: Missing security updates

1. Check Windows Update — Identifies missing patches.
2. Install updates — Fixes known vulnerabilities.
3. Restart computer — Applies updates fully.
4. Check update history — Ensures updates installed correctly.
5. Run update troubleshooter — Fixes update issues.
6. Free disk space — Updates require storage.
7. Disable VPN — May block updates.
8. Install manually — Alternative method if automatic fails.
9. Enable automatic updates — Prevents future issues.
10. Escalate if updates fail — Indicates deeper system issue.

Problem 6: Weak or reused password

1. Inform user — Explain risks of weak passwords.
2. Require password change — Forces stronger password.
3. Enforce password policy — Ensures complexity requirements.
4. Encourage passphrases — Easier and more secure than short passwords.
5. Enable MFA — Adds second layer of protection.
6. Check for reused passwords — Prevents cross-account breaches.
7. Recommend password manager — Helps generate/store strong passwords.
8. Educate user — Improves long-term security habits.
9. Document action — Maintains compliance record.
10. Monitor account — Ensures no compromise occurs.

Problem 7: Suspicious USB device used

1. Do not open files — Prevents malware execution.
2. Scan USB with antivirus — Detects threats.
3. Disable autorun — Prevents automatic execution.
4. Remove USB safely — Avoids data corruption.
5. Inform user — Educates on USB risks.
6. Check system logs — Identifies activity.
7. Run full system scan — Ensures no infection spread.
8. Block USB ports (if policy) — Prevents future use.
9. Document incident — Required for tracking.
10. Escalate if needed — If infection suspected.

Problem 8: Browser security warning

1. Do not proceed — Prevents accessing unsafe websites.
2. Check URL — Verify legitimacy.
3. Check certificate — Invalid certificates trigger warnings.
4. Try another browser — Helps confirm issue.
5. Check system date/time — Incorrect time causes certificate errors.
6. Clear browser cache — Fixes loading issues.
7. Disable VPN — VPN may interfere.
8. Check DNS — DNS issues can redirect traffic.
9. Inform user — Explain risks.
10. Escalate if legitimate site — May indicate larger issue.

Problem 9: Unauthorized software installed

1. Verify software — Confirm if it is allowed.
2. Uninstall program — Removes potential threat.
3. Scan system — Ensures no malware remains.
4. Check installation source — Identifies how it was installed.
5. Inform user — Prevents repeat behavior.
6. Restrict permissions — Prevent future installations.
7. Update antivirus — Detect similar threats.
8. Document incident — Maintains audit trail.
9. Monitor system — Ensure stability.
10. Escalate if suspicious — May indicate breach.

---

Problem 10: User shares sensitive data accidentally

1. Identify data — Determine what was exposed.
2. Stop sharing immediately — Prevent further exposure.
3. Revoke access — Remove permissions to shared data.
4. Inform user — Explain impact.
5. Notify security team — Required for compliance.
6. Check access logs — Identify who accessed data.
7. Secure affected accounts — Reset passwords if needed.
8. Document incident — Required for reporting.
9. Follow company policy — Ensures proper handling.
10. Monitor situation — Ensure no further impact.
