Problem 1: No internet connection

1. Check physical connection — Ensure the Ethernet cable is properly connected or Wi-Fi is enabled.
2. Verify Wi-Fi connection — Confirm the device is connected to the correct wireless network.
3. Restart the computer — Clears temporary network issues and resets adapters.
4. Restart router/modem — Fixes many temporary connectivity problems at the source.
5. Run `ipconfig` — Checks if the device has a valid IP address.
6. Ping default gateway — Confirms communication with the local network router.
7. Ping external IP (e.g., 8.8.8.8) — Verifies internet access without DNS involvement.
8. Check DNS settings — Incorrect DNS can block access to websites.
9. Reset network adapter — Reinitializes network configuration.
10. Test with another device — Determines if the issue is device-specific or network-wide.

Problem 2: Wi-Fi not connecting

1. Check Wi-Fi is enabled — Ensure the wireless adapter is turned on.
2. Verify password — Incorrect credentials will prevent connection.
3. Forget and reconnect to network — Clears corrupted connection profiles.
4. Restart Wi-Fi adapter — Resets wireless connection.
5. Restart router — Fixes temporary wireless issues.
6. Check signal strength — Weak signal can prevent connection.
7. Update network drivers — Fixes compatibility issues.
8. Disable airplane mode — Ensures wireless connections are allowed.
9. Check MAC filtering — Router may block unknown devices.
10. Test another network — Determines if issue is network-specific.

---

Problem 3: Wi-Fi connected but no internet

1. Check IP address — A 169.x.x.x address indicates DHCP failure.
2. Restart computer — Resets network configuration.
3. Restart router — Refreshes DHCP and network settings.
4. Renew IP address — Requests a new IP from the router.
5. Ping gateway — Confirms connection to local network.
6. Ping external IP — Checks if internet access exists.
7. Check DNS — DNS failure prevents website access.
8. Disable VPN — VPN can block internet if misconfigured.
9. Test another device — Determines if issue is widespread.
10. Contact ISP — Internet provider may have an outage.

Problem 4: VPN not working

1. Check internet connection — VPN requires stable internet access.
2. Verify login credentials — Incorrect username/password blocks access.
3. Restart VPN client — Clears temporary errors.
4. Check MFA — Multi-factor authentication may be failing.
5. Try another network — Some networks block VPN traffic.
6. Check firewall — Firewall may block VPN ports.
7. Update VPN client — Fixes bugs and compatibility issues.
8. Reinstall VPN software — Resolves corrupted installations.
9. Check VPN server status — Server may be down.
10. Review VPN logs — Helps identify specific connection issues.

Problem 5: Slow internet connection

1. Run speed test — Measures actual internet performance.
2. Close background applications — Reduces bandwidth usage.
3. Restart router — Fixes temporary slowdowns.
4. Move closer to router — Improves Wi-Fi signal strength.
5. Switch Wi-Fi band (2.4GHz/5GHz) — Reduces interference.
6. Use Ethernet cable — Provides more stable connection.
7. Check for network congestion — Multiple users can slow speeds.
8. Disable VPN — VPN can reduce speed.
9. Scan for malware — Malicious software may use bandwidth.
10. Check ISP status — Provider issues can cause slow speeds.

Problem 6: DNS issues (websites not resolving)

1. Try accessing via IP — Determines if DNS is the issue.
2. Run `nslookup` — Tests DNS resolution directly.
3. Flush DNS cache — Removes outdated DNS records.
4. Restart computer — Resets DNS resolver.
5. Change DNS server (e.g., 8.8.8.8) — Uses reliable external DNS.
6. Restart router — Refreshes DNS settings.
7. Check network settings — Ensures DNS is configured correctly.
8. Disable VPN — VPN may override DNS.
9. Test another device — Determines if issue is device-specific.
10. Contact ISP — DNS servers may be down.

Problem 9: Cannot access internal company resources

1. Check VPN connection — Internal access usually requires VPN.
2. Verify login credentials — Incorrect login blocks access.
3. Restart VPN client — Clears connection issues.
4. Check DNS — Internal resources rely on correct DNS.
5. Check firewall — May block internal services.
6. Restart computer — Resets connections.
7. Test another device — Helps isolate issue.

9. Contact IT for access — Permissions may need updating.

Problem 10: IP address conflict

1. Restart computer — Requests a new IP address.
2. Restart router — Resets DHCP assignments.
3. Release and renew IP — Forces a new IP allocation.
4. Check DHCP settings — Ensures automatic IP assignment.
5. Disable static IP — Prevents duplicate IP usage.
6. Check other devices — Identify conflicting device.
7. Scan network — Detect duplicate IP addresses.
8. Assign new IP manually — Temporarily resolves conflict.
9. Update network drivers — Fixes adapter issues.
10. Contact IT/network admin — Required if conflict persists.
