Problem 1: Cannot log in

1. Check password — Ensure the user is entering the correct password without typing errors.
2. Check Caps Lock — Verify that uppercase lock is not altering the password input.
3. Verify keyboard layout — Confirm the correct language layout is used (e.g., EN vs SE).
4. Check network/VPN — Ensure the device is connected to the required network or VPN.
5. Test another device — Helps determine if the issue is device-specific.
6. Try web login — Confirms whether the issue is limited to a specific application.
7. Check account status — Verify the account is not locked, disabled, or expired.
8. Restart device — Clears temporary glitches affecting login services.
9. Document error message — Capture the exact error for troubleshooting.
10. Escalate — Forward the issue with logs if unresolved.

---

Problem 2: Account locked

1. Check lock reason — Identify if too many failed login attempts caused the lock.
2. Unlock account — Restore access through the system or admin tools.
3. Reset password — Prevent repeated lockouts due to incorrect credentials.
4. Check saved credentials — Stored passwords in apps may cause repeated failures.
5. Check brute-force attempts — Ensure no suspicious activity triggered the lock.
6. Inform user — Explain what caused the lock and how to avoid it.
7. Restart device — Clears cached credentials that might retry login automatically.
8. Log out of all apps — Prevent background login attempts.
9. Document case — Record actions for tracking and auditing.
10. Escalate — If repeated lockouts occur.

---

Problem 3: MFA not working

1. Check device time — Incorrect time breaks authentication codes.
2. Test backup method — Use SMS/email if authenticator fails.
3. Sync authenticator app — Ensures generated codes are valid.
4. Restart phone — Fixes temporary device issues.
5. Check internet — Required for push-based authentication.
6. Clear app cache — Removes corrupted data.
7. Re-register MFA — Reset authentication setup completely.
8. Check account status — Ensure MFA is properly enabled.
9. Document issue — Record what failed and steps taken.
10. Escalate — If MFA service is down or misconfigured.

Problem 4: Access denied

1. Verify correct system — Ensure the user is accessing the right platform.
2. Check permissions — Confirm the user has the required access rights.
3. Check license — Some access depends on valid licenses.
4. Log out/in — Refresh session tokens.
5. Clear SSO session — Fix authentication token issues.
6. Try web version — Identify if issue is app-specific.
7. Test another device — Helps isolate the problem source.
8. Check policies — Access might be restricted by company rules.
9. Document error — Capture exact message/code.
10. Escalate — If permissions look correct but access fails.

Problem 5: Session keeps logging out

1. Clear cookies — Removes corrupted session data.
2. Clear cache — Fixes outdated browser data issues.
3. Try another browser — Helps isolate browser-specific problems.
4. Disable extensions — Extensions may interfere with sessions.
5. Check VPN — VPN can disrupt session stability.

7. Log out/in — Refresh authentication session.
8. Restart device — Clears temporary issues.
9. Document issue — Record frequency and pattern.
10. Escalate — If persistent across environments.
